# Dangerous Action Gates

Technical success does not automatically mean business, pedagogical, visual, or production acceptance. Each gate requires explicit evidence.

## Generation Gate

AI-generated text, images, or code may enter draft state. It becomes accepted only after human review for product fit, safety, and correctness.

## Retry Gate

Retrying a failed generation, test, payment action, or deployment is allowed only after identifying whether failure was transient, data-related, or logic-related.

## Schema Acceptance Gate

JSON validity means the file parses. Schema acceptance requires required fields, allowed values, and consistency with stage governance.

## Domain Acceptance Gate

Survival, medical, legal, and safety claims require qualified review before paid use.

## Lesson or Workflow Completion Gate

A user reaching the success screen means workflow completion. Learning completion additionally requires correct post-episode explanation or quiz evidence.

## Mastery or Authoritative Transition Gate

No user may be called "certified" or "master" by the product. The strongest allowed wording before expert review is "completed practical scenario".

## Rewards or Money Gate

Rewards, paid access, refunds, and purchase fulfillment must not be controlled only by client-side state.

## Notification Gate

User notifications require consent and clear purpose. No fear-based notification copy.

## Assembly Gate

A build artifact is not release-ready until build, docs validation, visual review, content review, and stage exit criteria pass.

## Downstream Gate

Any external publishing, payment, or email action must be reviewed for promise accuracy and privacy impact.

## Staging Gate

Staging acceptance requires:

- production-like build;
- no blocking console errors;
- first episode usable;
- docs validation proof current.

## Production Gate

Production deployment requires owner approval and stage contract GO.

## Production Accepted Gate

`production_accepted=true` is allowed only after:

- paid pilot flow tested;
- support path active;
- safety disclaimer reviewed;
- payment issues recovery path documented;
- owner records acceptance in decision log.

Current package status: `production_accepted=false`.
